Introduction

We noticed that non-professional recordings of instrument ensembles often contain unpleasant ambient noises. Inspired by this struggle, we decided to build a system to automatically remove ambient noises from instrumental audio recordings, restoring the clean sound of bells, acoustic guitar, violin, and other instruments from recordings contaminated with real-world noise, such as people-chatting, dog-barking, and air-conditioner noise, which are prevalent in musical recordings.

We plan to tackle the noise of people chattering from a dataset Sam found online (depositonce.tu-berlin.de). Once our model performs well enough on denoising this noise type from instrumentals, we will expand to other noise sources in the dataset (autos, coffee machine, dishwasher, washing machine, microwave, background music, radio, vacuum cleaner, TV, other minor ambient noises).

We first tried out simple filtering on the recordings to see the denoising effect, later attempting the same noise by applying multiple DSP techniques, including Analysis in a Different Basis, Frequency Domain Analysis (STFT), Sampling (Resampling).

Finally, we’ll train a deep learning model, specifically a convolutional neural network (CNN), to denoise our noisy instrumental audios. We chose to use a simple 2-level U-Net denoiser based on convolutional neural network (CNN) models. This approach first converts each noisy recording into a spectrogram — a 2D picture of how sound energy is distributed across frequency and time — and trains a convolutional neural network (CNN) that learns to generate denoised-version spectrograms.

Dataset preparation

The dataset consists of clean instrumentals, human speech noise files, and mixed audio of both. Every clean instrumental is randomly paired with a noise file and mixed with an arbitrary signal-to-noise ratio (SNR), then trimmed to desirable lengths.

Plot #1: Waveforms

Image Key: top: plot of clean violin example audio signal, middle: plot of people chattering example noise, bottom: plot of noise added to the clean audio

This shows three waveform plots, each representing a data type we will use in our project. We will be using clean instrumental audio, noise audio, and audio mixing the clean and the noise audio. We will mainly be focusing on eliminating the noise in subplot 3 and reverting it back to subplot 1. The audio sample used was of a violin recording, and the noise sample was of people chattering. We applied noise to the audio file by picking a random segment in the noise file equal to the length of the clean music file. We then multiplied this noise segment by the square root of the signal-to-noise ratio (snr) of the clean recording to the noisy recording. We then added that sound to the clean audio to obtain the final audio file with noise. All three files (clean audio, noise, noisy audio) can be found in the Work, Challenges, Learned tab.

How to noise the clean audio file was inspired by this source: https://www.mathworks.com/help/audio/ug/denoise-speech-using-deep-learning-networks.html

Plot #2: FFT

Image Key: left: FFT coefficient magnitude for CLEAN violin signal, right: FFT coefficient magnitude for NOISY violin signal

This shows the FFT coefficients of our clean data and noisy data (which mixed the clean and noise). In our project, we will use STFT as our input to the CNN model. And STFT is doing FFT for each frame. From the comparison, we observe additional spikes in the low-frequency range of the noisy data, which correspond to the synthesized noise we introduced.

Plot #3: STFT

Image Key: left: STFT Spectrogram for CLEAN violin signal, right: STFT spectrogram for NOISY violin signal

This shows the STFT of our noisy data. This will be our main input to the CNN model. The model will then learn from the spectrogram and do their best to denoise the noisy audio. From the comparison, it can be observced that the noisy data has higher power in the low-frequency range across all time periods.

Challenges

Our group is new to CNNs and Machine Learning models in general, so we have been researching scholarly articles and Matlab documentation to help us. Originally, we wanted to use audios recorded by ourselves. However, we soon found out the dataset required for training is large and we could not have so many audios prepared in a short time. Also, it is hard to create noisy data that uses the same clean data which we will use for our model's ground truth. Therefore, we switched to synthesizing noisy data instead of recording it by ourselves. Another thing is one of our goals is trying to eliminate multiple possible sources of noise, meaning that we will need a lot of training data and good time management.

What we Learned

We found that when processing audio signals, turning it into a spectrogram may produce a new perspective for analysis. The spectrogram can then be used with CNN which is normally an image processing model. This shows that Image processing models can also be used for 1 dimensional signals.
